# Лабораторная работа №5 - Отладка Python-проекта

- Исходный проект: лабораторная работа №4 ("симуляция библиотеки").
- Отчёт и доказательства: `report_lab5.md` + скриншоты в `assets/lab5/`.
- Инструмент: `debug_run.py` с переключаемыми сценариями.


## Цель
Цель работы - закрепить навыки работы с отладчиком: находить, анализировать и исправлять логические или runtime-ошибки в уже работающем проекте на Python.


## Тестовые сценарии отладки (`debug_run.py`)
Файл `debug_run.py` содержит набор сценариев (`case_1`…`case_5`), которые демонстрируют типовые проблемные участки проекта и служат точками входа для отладчика:

- `CASE = 1` - единичный запуск симуляции (`run_simulation`) с фиксированным seed.
- `CASE = 2` - создание библиотеки, добавление книги и поиск по жанру.
- `CASE = 3` - попытка удалить книгу через событие удаления (`remove_book_event`).
- `CASE = 4` - проверка того, что разные коллекции не разделяют один и тот же внутренний список.
- `CASE = 5` - последовательность событий (добавление, поиск по году/жанру) для локального анализа.

Изменить сценарий можно, установив `CASE` в нужное значение перед запуском (`python debug_run.py`).


## Проверка и отладка
1. Создайте виртуальное окружение и установите зависимости:
   ```
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
2. Выберите сценарий в `debug_run.py`, установите `CASE`, запустите отладчик.
3. После фиксации ошибок запустите:
   ```
   python debug_run.py
   ```
   чтобы убедиться, что выбранный сценарий выполняется без исключений.
4. Повторите для всех сценариев, отражённых в отчёте.


## Отчёт и подтверждения
- `report_lab5.md` содержит детальные описания найденных ошибок, шагов отладки, причин и исправлений.
- В отчёте указываются точки останова, значения переменных, кадры стека и скриншоты (`assets/lab5/`).
